# Specification documents

Place your **three** guiding documents in this folder so they are versioned with the repo and easy for contributors (and the AI) to find.

## Expected files

Use these names if possible (rename on copy is fine):

| # | Suggested filename | Typical content |
|---|--------------------|-----------------|
| 1 | `alphadock-architecture.docx` (or `.pdf`) | System architecture, services, deployment, endpoints overview |
| 2 | `alphadock-feature-spec.docx` (or `.pdf`) | Features, UI flows, tools, databases, docking lab design |
| 3 | `alphadock-repo-guide.docx` (or `.pdf`) | Monorepo layout, env vars, Makefile/CI, onboarding |

If your files use different names, that is fine—just keep all three here or tell the team the exact filenames.

## After you add them

Commit the files (or use Git LFS if they are very large), then ask for implementation work in chat with `@docs/specifications/…` or attach the relevant sections.

## Current status

This folder intentionally contains only this README until the specifications are added.
